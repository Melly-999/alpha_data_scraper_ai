# DEMO-004 CV and Portfolio Bullets

## Recruiter-Friendly Bullets
- Built a read-only AI trading terminal with an institutional dark UI and explicit safety posture.
- Designed Paper Sandbox Preview and Activity/Audit Rail views to support audit-first decision making.
- Added GET-only paper sandbox smoke validation to verify safe local demo flows.
- Kept the interface conservative: no live trading, no order controls, no broker execution UX.

## Technical Bullets
- React + Vite frontend for a terminal-style trading workspace.
- FastAPI backend with read-only paper sandbox preview/history endpoints.
- Safety-first terminal UX with visible dry-run, read-only, and live-orders-blocked status.
- Audit-oriented dashboard surfaces for broker posture, signals, and portfolio readouts.
- Local smoke script for validating demo routes and paper sandbox status.
- Designed to degrade safely when backend services are unavailable.
