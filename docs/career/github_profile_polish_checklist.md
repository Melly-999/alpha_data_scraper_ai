# GitHub Profile Polish Checklist

**Purpose:** Make the GitHub profile recruiter-readable without exposing private contact data or overstating experience.

## GitHub Bio

Use a short, honest bio:

```text
Self-taught Python / AI automation developer building safety-first fintech and workflow automation projects. FastAPI, React/TypeScript, supervised AI-assisted engineering.
```

Alternative:

```text
AI-assisted Python / automation developer. Building MellyTrade, a read-only fintech workstation with FastAPI, React, risk guardrails and audit-first UX.
```

## Pinned Repos

Pin in this order:

1. `alpha_data_scraper_ai` / MellyTrade - primary portfolio project.
2. Job Scanner Agent repo when created - AI automation / career tooling.
3. One small clean Python automation repo - demonstrates focused code quality.
4. Optional frontend/dashboard repo - if polished and public-safe.

## Profile README Structure

Recommended sections:

1. `Who I am` - self-taught Python / AI automation developer with customer operations background.
2. `What I am building` - MellyTrade as read-only fintech workstation portfolio project.
3. `What I can help with` - AI automation, technical support, FastAPI APIs, dashboard workflows, documentation.
4. `Featured project` - MellyTrade pitch, safety posture, stack and links.
5. `How I use AI tools` - supervised engineering workflow with human review.
6. `Currently learning` - FastAPI deeper, PostgreSQL, Docker, OpenAI Agents SDK, LangGraph.
7. `Contact` - use `[EMAIL]`, `[LINKEDIN]`, `[PORTFOLIO]` placeholders until private details are intentionally filled outside the repo.

Read-only AI-assisted fintech workstation: FastAPI + React terminal UI, risk guardrails, audit-first UX, dry-run only, no live trading.

## Screenshot Usage

Use 3-5 screenshots maximum on the profile:

- Terminal overview.
- AI Workspace.
- Risk guardrails.
- Broker read-only state.
- Audit/events feed.

Rules:

- Do not show private contact data.
- Do not show account IDs, credentials, tokens or real balances.
- Do not show order buttons or connect-live UX.
- Use captions that repeat read-only/dry-run positioning.

## What NOT To Claim

- Do not claim completed matura or degree unless confirmed.
- Do not claim commercial IT employment unless confirmed.
- Do not claim senior or mid-level experience.
- Do not claim MellyTrade is a commercial fintech platform.
- Do not claim live trading, broker execution, order placement or autonomous trading.
- Do not claim guaranteed trading results or financial advice.
- Do not say "AI wrote the code".

## Final Review Checklist

- Profile uses public-safe placeholders or public links only.
- MellyTrade is described as portfolio/read-only/dry-run.
- No private contact details are committed to any public repo.
- Pinned repos have clear README files and screenshots.
- Profile does not overstate education, commercial experience or trading capability.
- AI tooling is framed as supervised engineering assistance with human review.
