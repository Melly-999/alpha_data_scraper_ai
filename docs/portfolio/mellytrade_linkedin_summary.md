## 3. About section — longer version

I'm a junior backend developer focused on Python and FastAPI, building fintech
tools as portfolio projects. I care about designing systems where safety is an
architectural property, not a disclaimer.

My main project, MellyTrade, is a read-only AI trading terminal and portfolio
risk demo. It is deliberately *not* a live trading system: it's read-only,
dry-run, with live orders blocked and per-trade risk capped at 1%, enforced in
config, API, UI, and tests at the same time and served from a live safety
endpoint. The stack is FastAPI/Pydantic on the backend, React/TypeScript/Vite
on the frontend, with a mobile route and a Tauri desktop shell, deployed on
Render and Vercel.

Along the way I practiced real engineering discipline: CI (lint, type-check,
tests, e2e, security scans), public smoke-test evidence, deployment debugging,
and delivery through reviewed pull requests with documented review gates in an
AI-assisted workflow. I'm looking for junior backend / Python roles where I can
keep growing this kind of careful, well-documented engineering.
